# Development Guidelines

Rules for working on this codebase:

1. **Do not move files** — keep the existing project structure intact
2. **Scope changes** — when working on a page, only modify files relevant to that task
3. **Ask before big changes** — don't make sweeping modifications without approval
4. **Minimize backend edits** — avoid editing backend files unless necessary
5. **No unnecessary changes** — if it works, don't refactor it
6. **Keep docs updated** — update `README.md`, `probably.md`, and `APIs.md` after significant changes
7. **Follow existing patterns** — match the code style, naming, and architecture already in place

Reference `probably.md` and `APIs.md` for project architecture and API contracts.
